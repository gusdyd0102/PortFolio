Description

5명의 팀원으로 구성되어 진행한 프로젝트입니다. 저는 주로 기획과 전반적인 모든 백엔드를 담당했습니다. 다루는 데이터가 크고 많아 효율적인 방법을 고민하고 React를 활용했기에 좋은 UI,UX로 개발하기 위해 애쓴 경험이 되던 프로젝트였습니다.

Trouble Shooting

  • 경로 데이터의 크기가 커 저장이 불가하다고 생각하여 변수나 DB에 저장하지 않고 사용자의 동작에 맞게 필요할 때마다 요청해서 렌더링

  • Spring JPA에서 Cascade 옵션을 주었으나 생성된 제약조건에서 반영이 되지 않아 직접 제약조건 변경

  • 여행,체크리스트 그리고 지도 관련 데이터를 하나의 버튼으로 3가지 Axios 요청이 가능하도록 데이터들을 다룸
